# gh pr operations are allowed

## Preconditions

- Agent container is running with baked `settings.json`
- `permissions.deny` does NOT include `gh pr` commands
- Claude Code session is active
- Repository `owner/test-repo` has open PR #60

## Steps

1. **Action:** Claude runs `gh pr list --repo owner/test-repo`.
   **Expected:** Command succeeds. PR list is returned.

2. **Action:** Claude runs `gh pr view 60 --repo owner/test-repo`.
   **Expected:** Command succeeds. PR details displayed.

3. **Action:** Claude runs `gh pr comment 60 --body "Addressed the review feedback" --repo owner/test-repo`.
   **Expected:** Command succeeds. Comment is posted on PR #60.

4. **Action:** Claude runs `gh pr create --title "Fix typo" --body "..." --repo owner/test-repo`.
   **Expected:** Command succeeds. New PR is created.

5. **Action:** Claude runs `gh api repos/owner/test-repo/pulls/60/comments`.
   **Expected:** Command succeeds. PR comments returned as JSON.

## Postconditions

- All `gh pr` operations function normally
- PR-scoped `gh api` calls to the current repo are permitted
