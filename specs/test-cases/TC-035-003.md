# bot-authored comments are excluded from pr.commented events

## Preconditions

- `gitcheck` CLI is installed and running in poll mode
- `GITHUB_TOKEN` is set
- Repository `owner/test-repo` has open PR #15 with label `spec:architecture`
- Listener state has PR #15 with `CommentCount: 2`

## Steps

1. **Action:** `dependabot[bot]` posts a comment on PR #15 (comment count becomes 3). The latest comment's `user.login` is `dependabot[bot]`.
   **Expected:** During the next poll cycle, the poller fetches the latest comment and sees the author ends with `[bot]`. No `pr.commented` event is emitted.

2. **Action:** Verify listener state after the poll cycle.
   **Expected:** `CommentCount` for PR #15 is updated to 3 (count is tracked), but no event was dispatched.

3. **Action:** A human collaborator posts a comment on PR #15 (comment count becomes 4).
   **Expected:** During the next poll cycle, the poller fetches the latest comment, sees a non-bot author, and emits a `pr.commented` event with `CommentAuthor` set to the human's login.

## Postconditions

- Bot comments update the stored count but do not generate events
- Human comments following bot comments generate events normally
