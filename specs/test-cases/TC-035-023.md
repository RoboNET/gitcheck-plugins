# existing pr.review_submitted events continue to work unchanged

## Preconditions

- `gitcheck` CLI is installed with the updated code that includes `pr.commented` support
- Plugin.yaml contains both old and new rules:
  ```yaml
  - type: pr.review_submitted
    labels: ["spec:*"]
    verdict: changes_requested
    action: "/spec inbox"
  - type: pr.commented
    labels: ["spec:*"]
    action: "/spec inbox"
  ```
- Repository has open PR #80 with label `spec:architecture`

## Steps

1. **Action:** A reviewer submits a review with `changes_requested` verdict on PR #80.
   **Expected:** Listener emits a `pr.review_submitted` event with `verdict: changes_requested`. EventRouter matches the first rule and dispatches `/spec inbox`.

2. **Action:** Verify no `pr.commented` event is emitted for the review submission.
   **Expected:** Review submissions are a separate event type. Only `pr.review_submitted` is emitted.

3. **Action:** A reviewer posts a regular comment (not a review) on PR #80.
   **Expected:** Listener emits a `pr.commented` event (separate from the review). EventRouter matches the `pr.commented` rule.

## Postconditions

- `pr.review_submitted` and `pr.commented` are independent event types
- Adding `pr.commented` does not interfere with existing review event handling
