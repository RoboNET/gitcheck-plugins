# null COMMENT_BODY replaced with empty string in action template

## Preconditions

- `gitcheck` CLI is installed
- Plugin.yaml contains a rule with `{COMMENT_BODY}` in the action template
- A `pr.commented` event is emitted with `CommentBody: null` (fetch failed) and `CommentAuthor: null`

## Steps

1. **Action:** EventRouter processes the event and resolves `{COMMENT_BODY}` and `{COMMENT_AUTHOR}`.
   **Expected:** Both variables are replaced with empty string `""`. The action string has no literal `{COMMENT_BODY}` or `{COMMENT_AUTHOR}` text.

2. **Action:** Verify the action is still dispatched to the agent.
   **Expected:** Agent session starts with the action (minus the empty variable values). No error or crash from null variable substitution.

## Postconditions

- Null comment fields produce empty string substitutions
- Action dispatch is not blocked by null fields
