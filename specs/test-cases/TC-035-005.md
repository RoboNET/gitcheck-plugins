# CommentId used as deduplication discriminator in event ID

## Preconditions

- `gitcheck` CLI is installed
- `GITHUB_TOKEN` is set
- Repository `owner/test-repo` has open PR #25 with label `spec:documentation`
- Listener state is fresh (no processed events)

## Steps

1. **Action:** A collaborator posts comment ID 90001 on PR #25. Run listener for one poll cycle with `--json`.
   **Expected:** `pr.commented` event emitted with `CommentId: 90001`. Event ID in `processed_events` contains `90001` as discriminator.

2. **Action:** Stop and restart the listener (state persists). Run another poll cycle without any new comments.
   **Expected:** No `pr.commented` event emitted. Comment ID 90001 is already in processed set.

3. **Action:** A collaborator posts comment ID 90002 on PR #25. Run another poll cycle.
   **Expected:** A new `pr.commented` event emitted with `CommentId: 90002`. Event ID is distinct from the previous one.

## Postconditions

- Each comment ID produces a unique event ID
- Duplicate comment IDs are not re-emitted across restarts
