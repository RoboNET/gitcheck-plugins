# rate limit caps pr.commented to one event per PR per poll cycle

## Preconditions

- `gitcheck` CLI is installed and running in poll mode with `--json`
- `GITHUB_TOKEN` is set
- Repository `owner/test-repo` has open PR #20 with label `spec:implementation`
- Listener state has PR #20 with `CommentCount: 5`

## Steps

1. **Action:** Three different collaborators each post a comment on PR #20 between two poll cycles (comment count goes from 5 to 8).
   **Expected:** During the next poll cycle, the poller detects comment count changed (5 -> 8).

2. **Action:** Observe JSON output from the poll cycle.
   **Expected:** Exactly one `pr.commented` event is emitted for PR #20 (not three). The event contains the **latest** comment's body and author.

3. **Action:** Verify the event's `CommentBody` and `CommentAuthor`.
   **Expected:** Fields contain the text and login of the most recently created comment (by `created_at`), not the first or middle comment.

## Postconditions

- At most one `pr.commented` event per PR per poll cycle
- The event always carries the latest comment's context
