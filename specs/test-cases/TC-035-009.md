# graceful degradation when comment fetch fails

## Preconditions

- `gitcheck` CLI is installed and running in poll mode with `--json`
- `GITHUB_TOKEN` is set but has hit the GitHub API rate limit (403 responses)
- Repository `owner/test-repo` has open PR #42 with label `spec:test-cases`
- PR #42 comment count increased since last poll

## Steps

1. **Action:** Wait for the poller to detect the comment count change on PR #42 and attempt to fetch comments.
   **Expected:** Comment fetch API calls return 403 (rate limited). CLI logs a warning about the failed fetch.

2. **Action:** Observe the emitted event in JSON output.
   **Expected:** A `pr.commented` event is still emitted for PR #42, but with `CommentBody: null`, `CommentAuthor: null`. The event is not suppressed.

3. **Action:** Verify the agent session receives the event and handles null comment context.
   **Expected:** Agent falls back to reading comments via `gh pr view` or `gh api` in its session.

## Postconditions

- Comment fetch failure does not suppress the event
- `CommentBody` and `CommentAuthor` are null when fetch fails
- Listener continues operating normally after the failure
