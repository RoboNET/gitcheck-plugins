# pr.commented event emitted when comment count increases on matching PR

## Preconditions

- `gitcheck` CLI is installed
- `GITHUB_TOKEN` environment variable is set
- Repository `owner/test-repo` has open PR #10 with label `spec:requirements`
- PR #10 currently has 3 comments (2 issue + 1 review)
- `~/.gitcheck/listener-state.json` contains a snapshot for PR #10 with `CommentCount: 3`

## Steps

1. **Action:** A collaborator posts a new issue comment on PR #10 (total becomes 4).
   **Expected:** GitHub API `GET /repos/owner/test-repo/pulls` returns `comments: 3, review_comments: 1` for PR #10.

2. **Action:** Run `gitcheck listen --poll --repo owner/test-repo --interval 10s --json` and wait for one poll cycle.
   **Expected:** CLI emits a `pr.commented` event in JSON output with fields: `type: "pr.commented"`, `pr_number: 10`, `CommentAuthor` set to the collaborator's login, `CommentBody` containing the comment text.

3. **Action:** Send SIGTERM and inspect `~/.gitcheck/listener-state.json`.
   **Expected:** Snapshot for PR #10 now has `CommentCount: 4`.

## Postconditions

- Exactly one `pr.commented` event emitted for the comment count change
- Listener state updated with new comment count
