# plugins without pr.commented rules silently ignore comment events

## Preconditions

- `gitcheck` CLI is installed
- Plugin.yaml contains only `pr.merged` and `pr.review_submitted` rules (no `pr.commented` rule)
- Repository has open PR #85 with label `spec:requirements`

## Steps

1. **Action:** A collaborator posts a comment on PR #85. Wait for one poll cycle.
   **Expected:** Listener detects the comment count change and emits a `pr.commented` event internally.

2. **Action:** EventRouter attempts to match the event against plugin rules.
   **Expected:** No rules match `pr.commented`. Event is silently discarded. No errors or warnings in logs.

3. **Action:** Verify listener continues operating normally after the unmatched event.
   **Expected:** Subsequent events (`pr.merged`, `pr.review_submitted`) are still processed correctly.

## Postconditions

- Unmatched `pr.commented` events are a no-op
- No log pollution from unmatched events
- Other event types are unaffected
