# no pr.commented event for PRs without matching labels

## Preconditions

- `gitcheck` CLI is installed and running in poll mode with `--json`
- `GITHUB_TOKEN` is set
- Repository `owner/test-repo` has two open PRs:
  - PR #30 with label `spec:requirements` (matching)
  - PR #31 with label `bugfix` (non-matching)
- Listener state has both PRs with `CommentCount: 0`

## Steps

1. **Action:** A collaborator posts a comment on PR #31 (non-matching label). Wait for one poll cycle.
   **Expected:** Comment count for PR #31 updates in state, but no `pr.commented` event is emitted because `bugfix` does not match any `pr.commented` rule in plugin.yaml.

2. **Action:** A collaborator posts a comment on PR #30 (matching label). Wait for one poll cycle.
   **Expected:** A `pr.commented` event is emitted for PR #30 with the comment context.

## Postconditions

- Only PRs with labels matching plugin rules generate `pr.commented` events
- Comment count is tracked for all PRs regardless of labels
