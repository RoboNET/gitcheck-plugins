# old listener state files without CommentCount parse correctly

## Preconditions

- `gitcheck` CLI is installed with updated code supporting `pr.commented`
- `~/.gitcheck/listener-state.json` was written by a previous version that has no `CommentCount` field in PR snapshots:
  ```json
  {
    "last_poll_ts": "2026-04-01T10:00:00Z",
    "processed_events": ["pr.merged:owner/test-repo:42:abc123"],
    "pr_snapshots": {
      "42": { "head_sha": "abc123", "labels": ["spec:requirements"], "was_merged": true, "merged_at": "2026-04-01T09:55:00Z" }
    }
  }
  ```
  Note: no `comment_count` field in snapshot for PR #42.

## Steps

1. **Action:** Start `gitcheck listen --poll --repo owner/test-repo --interval 10s` with the old state file.
   **Expected:** CLI starts without errors. Old state file is parsed successfully. Missing `CommentCount` defaults to 0.

2. **Action:** Wait for one poll cycle where PR #42 has `comments: 3, review_comments: 1` (total 4).
   **Expected:** Poller detects comment count change (0 -> 4) and emits a `pr.commented` event for PR #42 (since it appears as a new comment increase from the default 0).

3. **Action:** Send SIGTERM and inspect state file.
   **Expected:** State file now includes `comment_count: 4` in the snapshot for PR #42. File is valid JSON.

## Postconditions

- Old state files without `CommentCount` are backward-compatible
- Missing field treated as 0, triggering a one-time catch-up event on first poll
