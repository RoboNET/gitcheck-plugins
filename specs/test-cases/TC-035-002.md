# comment count tracks both issue comments and review comments

## Preconditions

- `gitcheck` CLI is installed
- `GITHUB_TOKEN` is set
- Repository `owner/test-repo` has open PR #12 with label `spec:test-cases`
- PR #12 has `comments: 2` (issue) and `review_comments: 1` (inline review) per GitHub API
- Listener state has PR #12 with `CommentCount: 3`

## Steps

1. **Action:** A reviewer posts a new inline review comment on PR #12 (review_comments becomes 2).
   **Expected:** GitHub API returns `comments: 2, review_comments: 2` for PR #12 (total 4).

2. **Action:** Wait for one poll cycle with `--json` output.
   **Expected:** CLI detects comment count change (3 -> 4) and emits a `pr.commented` event for PR #12.

3. **Action:** A collaborator posts a new issue comment on PR #12 (comments becomes 3). Wait for next poll cycle.
   **Expected:** CLI detects comment count change (4 -> 5) and emits a second `pr.commented` event for PR #12.

## Postconditions

- Both issue comments and review comments contribute to comment count
- Each type of comment increase triggers a separate event
