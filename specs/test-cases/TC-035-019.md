# destructive gh api calls blocked by permissions deny

## Preconditions

- Agent container is running with baked `settings.json`
- `permissions.deny` includes `Bash(gh api -X DELETE*)`, `Bash(gh api --method DELETE*)`
- Claude Code session is active

## Steps

1. **Action:** Claude attempts `gh api -X DELETE repos/owner/test-repo`.
   **Expected:** Blocked by permissions deny. Command not executed.

2. **Action:** Claude attempts `gh api --method DELETE repos/owner/test-repo/branches/main/protection`.
   **Expected:** Blocked. Cannot remove branch protection via API.

3. **Action:** Claude attempts `gh api -X DELETE repos/owner/test-repo/hooks/123`.
   **Expected:** Blocked. Cannot delete webhooks.

4. **Action:** Claude runs `gh api repos/owner/test-repo/pulls/60` (GET, no method flag).
   **Expected:** Allowed. Default GET requests are not blocked.

## Postconditions

- DELETE method API calls are blocked regardless of endpoint
- GET API calls to the repo are allowed
