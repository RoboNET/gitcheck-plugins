# COMMENT_BODY not included in default spec-pipeline action

## Preconditions

- `gitcheck` CLI is installed with the spec-pipeline plugin
- Plugin.yaml default rule for `pr.commented` is:
  ```yaml
  - type: pr.commented
    labels: ["spec:*"]
    action: "/spec inbox"
  ```
  Note: action does NOT contain `{COMMENT_BODY}` or `{COMMENT_AUTHOR}`

## Steps

1. **Action:** A `pr.commented` event is routed through EventRouter.
   **Expected:** The action `/spec inbox` is dispatched as-is (no variable substitution needed since no variables are in the template).

2. **Action:** Verify that `/spec inbox` reads all unresolved comments itself via `gh`.
   **Expected:** The agent uses `gh pr view` or `gh api` to fetch the full comment history, not relying on the event's `CommentBody` field.

## Postconditions

- Default action delegates comment reading to `/spec inbox` rather than passing comment text in the prompt
- Custom plugins can opt into `{COMMENT_BODY}` by including it in their action template
