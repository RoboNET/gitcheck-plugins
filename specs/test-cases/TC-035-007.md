# poller fetches latest issue comment via GitHub API

## Preconditions

- `gitcheck` CLI is installed and running in poll mode
- `GITHUB_TOKEN` is set with sufficient permissions
- Repository `owner/test-repo` has open PR #40 with label `spec:requirements`
- PR #40 comment count increased since last poll

## Steps

1. **Action:** Wait for the poller to detect the comment count increase on PR #40.
   **Expected:** Poller calls `GET /repos/owner/test-repo/issues/40/comments?sort=created&direction=desc&per_page=1`.

2. **Action:** Also verify poller fetches inline review comments.
   **Expected:** Poller calls `GET /repos/owner/test-repo/pulls/40/comments?sort=created&direction=desc&per_page=1`.

3. **Action:** The issue comment has `created_at: 2026-04-02T10:05:00Z` and the review comment has `created_at: 2026-04-02T10:03:00Z`.
   **Expected:** The event's `CommentBody` and `CommentAuthor` come from the issue comment (more recent). `CommentId` matches the issue comment's ID.

## Postconditions

- The most recent comment by `created_at` is selected regardless of comment type
- Both API endpoints are called for changed PRs
