# gh auth login/logout/switch blocked but gh auth status allowed

## Preconditions

- Agent container is running with baked `settings.json`
- `permissions.deny` includes `Bash(gh auth login*)`, `Bash(gh auth logout*)`, `Bash(gh auth switch*)`
- Claude Code session is active

## Steps

1. **Action:** Claude attempts `gh auth login --with-token`.
   **Expected:** Blocked by permissions deny. Command not executed.

2. **Action:** Claude attempts `gh auth logout`.
   **Expected:** Blocked. Agent cannot log out of the configured GitHub account.

3. **Action:** Claude attempts `gh auth switch --user other-account`.
   **Expected:** Blocked. Agent cannot switch GitHub accounts.

4. **Action:** Claude runs `gh auth status`.
   **Expected:** Allowed. Command executes and shows the current auth status. This is read-only and useful for debugging.

## Postconditions

- Auth-modifying operations are blocked
- Read-only `gh auth status` is permitted
