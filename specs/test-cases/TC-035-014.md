# first-match rule applies when multiple pr.commented rules exist

## Preconditions

- `gitcheck` CLI is installed
- Plugin.yaml contains two `pr.commented` rules in order:
  ```yaml
  - type: pr.commented
    labels: ["spec:requirements"]
    action: "/spec review-requirements"
  - type: pr.commented
    labels: ["spec:*"]
    action: "/spec inbox"
  ```
- A `pr.commented` event for a PR with label `spec:requirements`

## Steps

1. **Action:** EventRouter processes the event.
   **Expected:** The first rule matches (`spec:requirements` matches `spec:requirements`). Action `/spec review-requirements` is dispatched.

2. **Action:** Verify the second rule is not also triggered.
   **Expected:** Only one action is dispatched per event (first-match semantics). `/spec inbox` is NOT dispatched.

## Postconditions

- First matching rule wins; subsequent rules for the same event type are skipped
