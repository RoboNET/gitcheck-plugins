# PreToolUse hook blocks dangerous gh subcommands

## Preconditions

- Agent container is running with `secret-guard.sh` registered as PreToolUse hook
- The hook is extended to detect dangerous `gh` subcommands
- Claude Code session is active

## Steps

1. **Action:** Claude attempts `gh repo create new-repo` via Bash tool.
   **Expected:** `secret-guard.sh` intercepts the tool call, detects `gh repo create`, and blocks it with a descriptive error message.

2. **Action:** Claude attempts `gh ssh-key add /tmp/key.pub`.
   **Expected:** Hook detects `gh ssh-key` and blocks the call.

3. **Action:** Claude attempts `gh auth login --with-token <<< "ghp_fake"`.
   **Expected:** Hook detects `gh auth login` (not `gh auth status`) and blocks.

4. **Action:** Claude attempts `gh api -X DELETE repos/owner/test-repo/collaborators/user`.
   **Expected:** Hook detects `gh api` with DELETE method on a non-PR endpoint and blocks.

5. **Action:** Claude runs `gh pr list`.
   **Expected:** Hook allows the command. `gh pr` is not in the blocked list.

6. **Action:** Claude runs `gh auth status`.
   **Expected:** Hook allows. `gh auth status` is explicitly exempted.

## Postconditions

- PreToolUse hook provides a second layer of defense beyond permissions deny
- Allowed commands pass through the hook without delay
