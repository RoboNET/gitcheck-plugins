# gh repo create blocked by permissions deny

## Preconditions

- Agent container is running with baked `settings.json`
- `permissions.deny` includes `Bash(gh repo create*)`, `Bash(gh repo delete*)`, `Bash(gh repo fork*)`
- Claude Code session is active inside the container

## Steps

1. **Action:** Claude attempts to use Bash tool to run `gh repo create evil-repo --public`.
   **Expected:** Blocked by permissions deny. Claude Code rejects the tool call before execution. Error indicates the command is not permitted.

2. **Action:** Claude attempts `gh repo delete owner/test-repo --yes`.
   **Expected:** Blocked by permissions deny. Command is not executed.

3. **Action:** Claude attempts `gh repo fork owner/other-repo`.
   **Expected:** Blocked by permissions deny. Command is not executed.

## Postconditions

- No repositories were created, deleted, or forked
- All three `gh repo` subcommands are blocked at the Claude Code permissions layer
