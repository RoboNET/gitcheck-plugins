# EventRouter matches pr.commented with label glob pattern

## Preconditions

- `gitcheck` CLI is installed
- Plugin.yaml contains:
  ```yaml
  - type: pr.commented
    labels: ["spec:*"]
    action: "/spec inbox"
  ```
- Three `pr.commented` events ready for routing:
  - Event A: PR labels `["spec:requirements"]`
  - Event B: PR labels `["spec:test-cases", "priority:high"]`
  - Event C: PR labels `["bugfix"]`

## Steps

1. **Action:** EventRouter processes Event A.
   **Expected:** Matches the `pr.commented` rule (label `spec:requirements` matches glob `spec:*`). Action `/spec inbox` is dispatched.

2. **Action:** EventRouter processes Event B.
   **Expected:** Matches the rule (label `spec:test-cases` matches glob `spec:*`). Action dispatched.

3. **Action:** EventRouter processes Event C.
   **Expected:** No match. `bugfix` does not match `spec:*`. No action dispatched, no error logged.

## Postconditions

- Label glob matching works identically for `pr.commented` as for other event types
- Non-matching events are silently skipped
