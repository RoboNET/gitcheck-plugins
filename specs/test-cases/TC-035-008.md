# review comment selected when it is more recent than issue comment

## Preconditions

- `gitcheck` CLI is installed and running in poll mode with `--json`
- `GITHUB_TOKEN` is set
- Repository `owner/test-repo` has open PR #41 with label `spec:architecture`
- PR #41 comment count increased since last poll
- Latest issue comment: `created_at: 2026-04-02T09:00:00Z`, body: "old comment"
- Latest review comment: `created_at: 2026-04-02T09:05:00Z`, body: "inline fix needed"

## Steps

1. **Action:** Wait for one poll cycle and observe the emitted event.
   **Expected:** `CommentBody` is "inline fix needed" (the review comment, which is more recent). `CommentAuthor` is the review comment author's login.

## Postconditions

- The comparison uses `created_at` timestamps to determine the latest comment
- Review comments and issue comments are treated equally
