# no extra API calls for PRs with unchanged comment count

## Preconditions

- `gitcheck` CLI is installed and running in poll mode
- `GITHUB_TOKEN` is set
- Repository `owner/test-repo` has open PRs #50 and #51 with `spec:*` labels
- Listener state: PR #50 `CommentCount: 5`, PR #51 `CommentCount: 3`
- Since last poll: no new comments on either PR

## Steps

1. **Action:** Wait for one poll cycle and monitor HTTP requests made (via debug logging or network trace).
   **Expected:** Poller calls the Pulls list API to get current PR state, but does NOT call the issue comments or review comments endpoints for PR #50 or #51.

2. **Action:** Verify no `pr.commented` events in output.
   **Expected:** No events emitted. Comment counts remain unchanged in state.

## Postconditions

- Comment detail endpoints are only called when comment count differs from stored state
- Unchanged PRs incur zero additional API calls beyond the list endpoint
